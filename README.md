# Nexacraft Download Site

A self-contained static landing page for downloading Nexacraft. Pure HTML/CSS — no build step, no dependencies.

## Folder contents

```
website/
├── index.html      # the landing page
├── style.css       # all styling
├── assets/
│   ├── nexacraft-title.png    # logo
│   ├── favicon.png
│   ├── panorama-seamless.png  # scrolling background
│   ├── screenshot.png         # hero image
│   └── fonts/                 # Minecraft webfonts
├── downloads/                 # generated — the actual .dmg files
└── README.md       # this file
```

`downloads/` is generated by `npm run site:publish` and is gitignored. Don't edit it by hand.

---

## Updating the site to a new version of the game

One command:

```bash
npm run site:publish
```

This will:

1. Run `vite build` + `electron-builder --mac` to produce fresh `.dmg` files in `release/`.
2. Copy them into `website/downloads/`.
3. Update the file-size labels in `website/index.html` to match the new sizes.

After that, run `npm run site:serve` (or just refresh your browser) to preview at <http://localhost:8000>.

## Previewing locally

```bash
npm run site:serve
# opens at http://localhost:8000
```

Or open `website/index.html` directly in a browser (download buttons may not trigger a real download via the `file://` protocol — use the server for that).

## Deploying to a real host

The `downloads/` folder is ~470 MB, which exceeds the file size limits on most free static hosts:

| Host | Per-file limit |
|---|---|
| GitHub Pages | 100 MB |
| Netlify (free) | 100 MB |
| Vercel (free) | 100 MB |
| Cloudflare Pages | 25 MB |

**So when you go live, the `.dmg` files need to live somewhere else** and the buttons should link to that "somewhere else." The recommended setup is **GitHub Releases** (free, 2 GB per file, fast CDN).

### Step 1 — Put the binaries on GitHub Releases

1. Push your project to a GitHub repo (if it isn't already).
2. On GitHub, go to your repo → **Releases** → **Draft a new release**.
3. Tag version: `v1.0.0-alpha`. Title: `Nexacraft v1.0.0 Alpha`.
4. Drag your built binaries from `release/` into the upload area:
   - `Nexacraft-1.0.0-arm64.dmg` (Apple Silicon)
   - `Nexacraft-1.0.0.dmg`       (Intel Mac)
   - (later) `Nexacraft Setup 1.0.0.exe` (Windows)
   - (later) `Nexacraft-1.0.0.AppImage`  (Linux)
5. Click **Publish release**.

Each file now has a permanent URL like:
```
https://github.com/<USER>/<REPO>/releases/download/v1.0.0-alpha/Nexacraft-1.0.0-arm64.dmg
```

### Step 2 — Swap the URLs in `index.html`

Find the 2 `download-card` blocks and change:

```html
<a class="download-card" href="downloads/Nexacraft-1.0.0-arm64.dmg" download>
```

To:

```html
<a class="download-card" href="https://github.com/<USER>/<REPO>/releases/download/v1.0.0-alpha/Nexacraft-1.0.0-arm64.dmg">
```

You can leave the `downloads/` folder in place; it just won't be referenced anymore.

### Step 3 — Deploy the site folder

Once the URLs point at GitHub, the `website/` folder is small (~10 MB) and any free static host will work:

- **Netlify drag-and-drop** (easiest): go to <https://app.netlify.com/drop>, drag the `website/` folder.
- **Cloudflare Pages**: <https://pages.cloudflare.com> — connect your repo.
- **GitHub Pages**: Repo → **Settings → Pages** → set source to `/website`.
- **Vercel**: <https://vercel.com/new> — import the folder.

All four are free, all auto-issue HTTPS, all accept custom domains.

### Step 4 — Custom domain

When you're ready to buy a domain:
- **Cloudflare Registrar** is cheapest (no markup), ~$10/yr for a `.com`. Other options: Namecheap, Porkbun.
- Possible names: `nexacraft.com`, `nexacraft.gg`, `playnexacraft.com`, `nexusgames.gg`.
- Add the domain in your host's dashboard — they walk you through the DNS records to set.

## Tips

- The `download` attribute on the local `<a>` tags forces the browser to download the file rather than try to open it inline. Once you switch to GitHub Releases URLs, you can leave `download` off — GitHub's CDN sends the right `Content-Disposition` header automatically.
- The screenshot at the top of the page is `assets/screenshot.png` — swap it out anytime for a different one.
- Need a new image generated? Just ask me to make one.
